# codingtest-study

## ✅ 참여 방법
1. 이 저장소를 `fork` 한다.
2. 생성된 원격 저장소에 `github ID`로 폴더를 생성한다.
3. 생성된 폴더에 자신의 소스코드를 `push` 한다.
4. 이때 `commit 규칙`을 지키도록 한다!
5. 원본 저장소로 `Pull Request`를 한다.
6. 다른 사람들의 PR을 보고 자유롭게 코드리뷰를 한다.


## ✅ commit 컨벤션
- commit 태그: [목차명]_문제번호_문제이름
- 문제 번호가 없을 경우 *로 표기

💡 **commit 예시**

`feat: [배열]_1000_A+B`

`feat: [배열]_*_A+B`

💡 **commit 태그**
| 태그 | 설명 |
|------|---|
| feat| 새로운 기능 추가 |
| fix | 버그 수정 |
| refactor | 코드 리팩토링 |
| comment	| 주석 추가(코드 변경 X) 혹은 오타 수정 |
| docs | README와 같은 문서 수정 |
| rename | 파일, 폴더명 수정 혹은 이동 |
